# Отчёт по анализу систем проекта Vitruvium

## Система: Dice (Броски кубов)

**Тип:** Core / Utility

**Описание:**
Реализует механику бросков кастомных кубов системы (1-3: провал, 4-5: 1 успех, 6: 2 успеха), поддержку пулов кубов, преимуществ/помех (перебросы) и "удачливых" бросков (полный переброс пула).

**Файлы:**
- `module/rolls.js` (основная логика)
- `module/dv-die.js` (определение типа куба)
- `module/combat.js` (дубликат логики `rollPool`)
- `module/initiative.js` (дубликат логики `rollPool`)
- `module/dice-so-nice.js` (интеграция 3D кубов)

**Ключевые функции:**
- `rollSuccessDice()` / `rollPool()`
- `dvSuccesses()` / `countSuccesses()`
- `applyReroll()`
- `pickIndex()`

**Используемые данные:**
- `pool` (размер пула)
- `luck` / `unluck` (счётчики преимуществ)
- `fullMode` (adv/dis/normal)

**Зависимости:**
- Наследует от `Roll` (Foundry VTT)
- Использует `effects.js` для получения модификаторов

**Проблемы:**
- **Критическое дублирование:** Логика вычисления успехов и перебросов реализована независимо в `rolls.js`, `combat.js` и `initiative.js`.
- **Разрозненность:** Нет единой точки входа для всех бросков; каждый модуль "варит" свой бросок по-своему, что ведёт к багам (например, разные реализации `dvSuccesses`).

**Оценка качества:**
- Связанность: Low (логика размазана)
- Дублирование: Yes (High)
- Централизация: Low

## Система: Effects (Система эффектов и модификаторов)

**Тип:** Data / Core

**Описание:**
Централизованная система для определения, нормализации и сбора модификаторов от предметов, способностей и состояний. Рассчитывает итоговые значения атрибутов и бонусов к броскам.

**Файлы:**
- `module/effects.js` (определения и логика расчёта)

**Ключевые функции:**
- `normalizeEffects()`
- `collectEffectTotals()`
- `getEffectiveAttribute()`
- `getGlobalRollModifiers()`

**Используемые данные:**
- Массив `EFFECT_TARGETS`
- Объект `system.effects` в документах Actor/Item

**Зависимости:**
- Используется во всех боевых и интерфейсных системах

**Проблемы:**
- **Отсутствие абстракции:** Система только считает "цифры", но не предоставляет API для автоматического применения (например, "применить эффект X на актора Y"). Это делается вручную в других местах.

**Оценка качества:**
- Связанность: High
- Дублирование: No
- Централизация: High

## Система: State Duration (Длительность состояний)

**Тип:** Glue / Core

**Описание:**
Отвечает за тиканье длительности состояний (State) в бою, удаление истекших эффектов и применение периодического урона/лечения (DoT/HoT).

**Файлы:**
- `module/state-duration.js`
- `module/state-library.js` (хранилище шаблонов)

**Ключевые функции:**
- `registerStateDurationHooks()`
- `tickActorOwnedStates()`
- `applyOverTimeEffectsForActor()`

**Зависимости:**
- Зависит от `effects.js` (для поиска DoT)
- Зависит от `Hooks` (Foundry VTT)

**Проблемы:**
- **Смешивание ответственности:** Модуль сам ищет эффекты, сам обновляет HP, сам удаляет предметы. Тесно связан с хуками боя.

**Оценка качества:**
- Связанность: Medium
- Дублирование: No
- Централизация: High

## Система: Combat (Боевая логика)

**Тип:** Core / UI

**Описание:**
"God-object" система, управляющая процессами атаки, защиты, расчёта урона и отображения карточек в чате. Обрабатывает взаимодействие между атакующим и защищающимся через сокеты.

**Файлы:**
- `module/combat.js`

**Ключевые функции:**
- `startWeaponAttackFlow()`
- `startAbilityAttackFlow()`
- `renderAttackCard()`

**Проблемы:**
- **Размазанность:** Содержит в себе всё — от математики кубов до генерации HTML-строк и логики сокетов.
- **Дублирование:** Имеет локальные копии функций `clamp`, `num`, `dvSuccesses`, которые есть в других файлах.

**Оценка качества:**
- Связанность: Low (смешано всё)
- Дублирование: Yes
- Централизация: Medium (один файл, но слишком большой)

## Потенциальная система: Core Utilities (Общие утилиты)

**Описание:**
Набор базовых функций для работы с числами, строками и безопасностью (HTML escape).

**Где найдено:**
- `character-sheet.js`
- `combat.js`
- `rolls.js`
- `initiative.js`
- `effects.js`
- `npc-sheet.js`
- `ability-sheet.js`

**Почему это система:**
- Функции `clamp`, `num` (или `toNumber`), `esc` (или `escapeHtml`) повторяются почти в каждом файле.
- Выполнение одинаковой функции (нормализация данных).

**Рекомендация:**
- Вынести все вспомогательные функции в отдельный файл `module/utils.js` и импортировать их оттуда.

---

## Потенциальная система: Chat UI (Интерфейс чата)

**Описание:**
Логика формирования визуальных карт для чата.

**Где найдено:**
- `combat.js`
- `rolls.js`
- `initiative.js`

**Почему это система:**
- Везде вручную собирается HTML-строками `v-card`, `v-box`, `v-details`.
- Стиль карточек (CSS классы) жёстко зашит в JS код в разных местах.

**Рекомендация:**
- Использовать Handlebars шаблоны для всех чат-карт (вынести из JS в `.hbs` файлы).
- Создать центральный класс `ChatManager` для отправки стандартизированных сообщений.
