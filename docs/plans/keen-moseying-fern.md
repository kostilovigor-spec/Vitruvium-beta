# План: Способы наложения состояний и фикс переключения вкладок

## Контекст
При выборе состояния или атрибутов в секции "Накладываемое состояние" на вкладке "Эффекты" ability-sheet происходит нежелательное переключение вкладки. Также нужно добавить выбор способа наложения состояния: "На себя", "Цель: без проверки", "Цель: соревнование".

## Изменения

### 1. Фикс переключения вкладок (ability-sheet.js)
В `activateListeners` — добавить `ev.preventDefault()` и `ev.stopPropagation()` на все select-ы внутри вкладки эффектов, чтобы change на них не вызывал переключение вкладки. Проблема в том, что Foundry `submitOnChange` при change на select-ах вызывает re-render, который сбрасывает активную вкладку.

**Решение:** Сохранять активную вкладку способности (`v-ability-tabs`) в `this._abilityTab` и восстанавливать при render. Добавить отслеживание radio кнопок вкладок.

### 2. Новое поле `system.contestApplyMode` (ability-sheet)
Три значения:
- `"self"` — На себя (состояние накладывается на кастера, без проверки)
- `"targetNoCheck"` — Цель: без проверки (состояние накладывается на цель без соревновательного броска)
- `"targetContest"` — Цель: соревнование (текущее поведение — оба бросают)

По умолчанию: `"targetContest"` (обратная совместимость).

### 3. Файлы и изменения

#### `templates/item/ability-sheet.hbs` (строки ~212-253)
- Перед блоком выбора атрибутов состязания добавить select для `system.contestApplyMode`
- Обернуть строки атрибутов кастера/цели в div с классом, который скрывается через CSS/JS когда mode != "targetContest"
- При mode "self" или "targetNoCheck" скрыть оба select атрибутов

#### `module/ability-sheet.js`
- Добавить handler `$contestApplyMode.on("change")` для немедленного сохранения
- Сохранять и восстанавливать активную вкладку (`this._abilityTab`)
- В `getData()` добавить `vitruvium.contestApplyMode`
- Показать/скрыть блок атрибутов состязания в зависимости от mode

#### `module/combat.js` — `startAbilityAttackFlow()` (строки 1788-2015)
- Читать `contestApplyMode` из ability
- Если `"self"`:
  - Не требовать таргет
  - Не делать contest бросок
  - Вызвать `replaceStateFromTemplate(attackerActor, ...)` на кастера
  - Вывести карточку в чат без кнопок сопротивления
- Если `"targetNoCheck"`:
  - Требовать таргет
  - Не делать contest бросок ни кастера ни цели
  - Вызвать `replaceStateFromTemplate(defender, ...)` сразу
  - Вывести карточку в чат без кнопки сопротивления
- Если `"targetContest"` — текущее поведение (без изменений)

#### `module/character-sheet.js` — `use-ability` handler (строки ~837-884)
- Учесть `contestApplyMode` при определении `useAsAttack`
- Для mode `"self"` — не передавать в `startAbilityAttackFlow`, а обработать прямо здесь: наложить состояние на кастера и вывести карточку

### 4. Порядок реализации
1. Фикс вкладок в ability-sheet (JS + HBS)
2. Добавить select `contestApplyMode` в шаблон
3. Handler в ability-sheet.js
4. Логика в combat.js
5. Логика в character-sheet.js

### 5. Проверка
- Открыть способность → вкладка "Эффекты" → менять select-ы → вкладка не должна переключаться
- Выбрать "На себя" → использовать → состояние на кастере, нет проверки
- Выбрать "Цель: без проверки" → использовать на цели → состояние применяется сразу
- Выбрать "Цель: соревнование" → старое поведение с бросками
