# Changelog

Все заметные изменения Vitruvium документируются здесь.

## [Unreleased]
### Добавлено
- **Унификация интерфейса урона**:
    - Удалены лишние шепотные сообщения ГМу о применении урона.
    - Кнопки применения урона теперь динамически внедряются в основную карточку атаки в список целей.
    - Кнопки видны только ГМу и появляются в том числе при промахе.
- **Исправление характеристик NPC**:
    - Исправлено расхождение в структуре данных (у NPC характеристики хранились как объекты, а не числа), из-за чего при бросках пулы кубов сбрасывались до 1.
    - Унифицирована логика получения характеристик через `getEffectiveAttribute` во всех ключевых местах (лист NPC, атаки оружием, защита).

### Изменено
- Боевая математика: урон без разницы успехов; броня режет только успехи атаки; уклонение = hit по atk>def.
- Блок: активная защита (defS) снижает базовый урон, броня режет только успехи атаки; добавлен выключаемый бонус брони к блоку.

### Добавлено
- Поля монет (бр/ср/зл) под бронёй и скоростью на листе персонажа.
- Чекбокс «Это атака» для способностей: при выключении не бросается атака, только dV урона/спаса.

### Исправлено
- «Это атака» показывается только в режиме редактирования (как «Активно»).
- Описание предметов/способностей/навыков сохраняется при закрытии листа и выборе иконки.
- По умолчанию «Это атака» выключено; в не‑редактируемом режиме отображается статус «Это атака: Да/Нет».

## [0.1.1]
### Добавлено
- Кастомная кость dV (d6) и правила успехов 1–3/4–5/6.
- Листы актёров: персонаж и NPC.
- Листы предметов: ability, item, skill, state.
- Система эффектов с глобальными модификаторами бросков.
- Боевой поток атака/защита/урон с GM‑resolve и скрытыми сообщениями.
- Переопределение инициативы с поддержкой преимуществ/помех.
- Интеграция Dice So Nice (пресет dV).
- Интеграция Automated Animations (при наличии модуля).
- Встроенные тесты (запуск через `game.vitruvium.runTests()`).
